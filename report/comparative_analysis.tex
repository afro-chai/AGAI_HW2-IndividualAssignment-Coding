\begin{abstract}
This report compares two graded behavioral strategies---\textbf{News Sentiment Follower} and \textbf{Volatility Averse}---plus an extension \textbf{Moral Trader} agent, implemented with Microsoft AutoGen and local inference (Ollama). All agents consume the same structured market JSON per ticker. The default equity set is \textbf{ten} U.S.-listed names spanning defense primes, AI analytics, semiconductors, \textbf{AFRICOM-relevant} energy and gold (TotalEnergies, Eni, Barrick), cybersecurity, and crude tankers---chosen to stress headline density and heterogeneous volatility (see \texttt{report/DEFENSE\_GEOPOLITICS\_UNIVERSE.md}). After each full run, quantitative claims in this PDF should be reconciled with \texttt{outputs/summary.json}, per-ticker \texttt{outputs/*.json}, and \texttt{outputs/backtest.json}. Expect roughly \textbf{50--90 minutes} wall time on a laptop for ten tickers (40 Ollama generations plus Alpha Vantage pacing).
\end{abstract}

\section{Strategy Selection and Rationale}

\textbf{Graded pair.} \textbf{News Sentiment Follower} vs.\ \textbf{Volatility Averse}: external narrative and sentiment (when news exists) versus measurable path risk (volatility, drawdowns, ATR).

\textbf{Defense--geopolitics framing.} The ticker universe is documented in \texttt{report/DEFENSE\_GEOPOLITICS\_UNIVERSE.md}. The default run uses symbols expected to carry \textbf{defense budget}, \textbf{regional conflict}, and \textbf{energy security} headlines---including \textbf{three AFRICOM-heavy picks} (\textbf{TTE}, \textbf{E}, \textbf{GOLD}) for Africa/MENA oil and conflict-adjacent mining narratives.

\textbf{Extension.} \textbf{Moral Trader} adds a third parallel branch (narrative opacity / accountability), evaluated only after A and B finish for the same ticker.

\section{System Architecture}

\subsection{Orchestration logic}
\begin{itemize}
  \item \textbf{Market data (no LLM):} \texttt{src/market\_data.py} pulls yfinance history and derived features; optionally calls Alpha Vantage \texttt{NEWS\_SENTIMENT}. API key: \texttt{ALPHAVANTAGE\_API\_KEY} in repo-root \texttt{.env} and/or gitignored \texttt{secrets/alphavantage\_api\_key.txt} (see \texttt{secrets/README.md}).
  \item \textbf{Strategy agents (LLM):} three \texttt{AssistantAgent} instances run in parallel per ticker with structured outputs (decision, confidence, justification).
  \item \textbf{Isolation:} strategies receive the same JSON context and do not see each other's outputs before the evaluator.
  \item \textbf{Evaluator (LLM):} compares outputs and writes consensus vs.\ divergence analysis.
  \item \textbf{Persistence:} per-ticker JSON under \texttt{outputs/}, plus \texttt{summary.json} and optional \texttt{backtest.json}.
\end{itemize}

\section{Patterns of Agreement and Disagreement}

\textbf{Reading \texttt{summary.json}.} With ten tickers, \texttt{total\_agreements} $+$ \texttt{total\_disagreements} $= 10$ (each row is one ticker). A higher disagreement count is \emph{more likely} with this basket than with a consumer-staple mix, but \textbf{not guaranteed}: local LLM conservatism and similar realized-vol regimes can still produce unanimous HOLD.

\textbf{Alpha Vantage at scale.} Ten tickers imply ten \texttt{NEWS\_SENTIMENT} requests per full pass; free-tier pacing may leave some tickers with empty \texttt{feed} while others populate---use \texttt{ALPHAVANTAGE\_MIN\_INTERVAL\_SEC} and/or \texttt{--ticker-workers 1} if you need more uniform news coverage.
