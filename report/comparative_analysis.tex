\begin{abstract}
This report compares two graded behavioral strategies---\textbf{News Sentiment Follower} and \textbf{Volatility Averse}---plus an extension \textbf{Moral Trader} agent, implemented with Microsoft AutoGen and local inference (Ollama). All agents consume the same structured market JSON per ticker. The default equity set is \textbf{five} U.S.-listed names: \textbf{PLTR}, \textbf{TTE}, \textbf{GOLD}, \textbf{LMT}, and \textbf{FRO}---see \texttt{report/DEFENSE\_GEOPOLITICS\_UNIVERSE.md}. Each full \texttt{python -m src.main} run issues \textbf{five} \texttt{NEWS\_SENTIMENT} calls (see \texttt{docs/ALPHA\_VANTAGE.md}); the historical backtest uses \textbf{zero} AV calls. The \textbf{2026-04-12} live pass (after \texttt{git pull}) produced real \textbf{disagreement}: \texttt{outputs/summary.json} reports \texttt{total\_agreements} $= 2$ and \texttt{total\_disagreements} $= 3$ when all three agents must align (unanimous only on \textbf{GOLD} and \textbf{LMT}). Table~\ref{tab:results} copies decisions and confidences from that run. The heuristic scorecard (Table~\ref{tab:backtest}) matches \texttt{outputs/backtest.json}: \textbf{115} weekly rows across the basket; \texttt{overall\_hit\_rate\_news} $= 0.4348$, \texttt{overall\_hit\_rate\_volatility} $= 0.2522$, \texttt{overall\_winner = news\_sentiment\_heuristic}.
\end{abstract}

\section{Strategy Selection and Rationale}

\textbf{Graded pair.} \textbf{News Sentiment Follower} vs.\ \textbf{Volatility Averse}: external narrative and sentiment (when news exists) versus measurable path risk (volatility, drawdowns, ATR).

\textbf{Defense--geopolitics framing.} The ticker universe is documented in \texttt{report/DEFENSE\_GEOPOLITICS\_UNIVERSE.md}. The default run uses a \textbf{small, diverse} basket with explicit \textbf{AFRICOM} hooks on \textbf{TTE} (Africa--MENA oil and LNG) and \textbf{GOLD} (Mali / DRC gold exposure narratives), plus U.S.\ defense software, a prime, and tanker beta.

\textbf{Extension.} \textbf{Moral Trader} adds a third parallel branch (narrative opacity / accountability), evaluated only after A and B finish for the same ticker.

\section{System Architecture}

\subsection{Orchestration logic}
\begin{itemize}
  \item \textbf{Market data (no LLM):} \texttt{src/market\_data.py} pulls yfinance history and derived features; optionally calls Alpha Vantage \texttt{NEWS\_SENTIMENT} once per ticker. API key: \texttt{ALPHAVANTAGE\_API\_KEY} in repo-root \texttt{.env} and/or gitignored \texttt{secrets/alphavantage\_api\_key.txt} (see \texttt{secrets/README.md}).
  \item \textbf{Strategy agents (LLM):} three \texttt{AssistantAgent} instances run in parallel per ticker with structured outputs (decision, confidence, justification).
  \item \textbf{Isolation:} strategies receive the same JSON context and do not see each other's outputs before the evaluator.
  \item \textbf{Evaluator (LLM):} compares outputs and writes consensus vs.\ divergence analysis.
  \item \textbf{Persistence:} per-ticker JSON under \texttt{outputs/}, plus \texttt{summary.json} and optional \texttt{backtest.json}.
\end{itemize}

\section{Patterns of Agreement and Disagreement}

\textbf{Reading \texttt{summary.json}.} For 2026-04-12, \texttt{total\_agreements} $= 2$ and \texttt{total\_disagreements} $= 3$ under the rule ``agree only if all three agents share the same decision.'' \textbf{GOLD} is unanimous \textbf{SELL} (graded agents at 8, Moral at 5); \textbf{LMT} is unanimous \textbf{HOLD} with a tight confidence band (spread 1). \textbf{PLTR} splits HOLD vs.\ SELL/Sell; \textbf{TTE} splits BUY vs.\ HOLD/HOLD; \textbf{FRO} splits HOLD vs.\ SELL/HOLD.

\textbf{Confidence spread.} Widest spreads are \textbf{PLTR} and \textbf{GOLD} (7 points): Moral Trader assigns confidence 1 on PLTR while Strategy~B is at 8; on GOLD Strategy~B reports confidence 1 while Strategy~A is at 8.

\textbf{Alpha Vantage budget.} Five tickers $\Rightarrow$ five news calls per run. In this pass, feeds were still mostly empty (\texttt{articles\_count = 0}) with mixed \texttt{note} text (daily-cap wording vs.\ pacing reminder on TTE), which makes Strategy~A's behavior more model-dependent and prone to hallucinated ``sentiment'' unless prompts ground strictly on provided fields.

\section{Failure or Surprise Case}

\textbf{Hallucination risk (TTE):} Strategy~A cites a ``positive sentiment score (0.05)'' while \texttt{avg\_sentiment\_score} is \texttt{null} and \texttt{articles\_count = 0}---a clear violation of ground-truth fields that the rubric would mark as a responsible-AI issue.

\textbf{Operational note:} empty feeds plus \texttt{news\_features.note} still trigger Moral Trader's ``empty news packet'' narrative on several names; \texttt{src/market\_data.py} scrubs echoed key material from Alpha Vantage strings before JSON is written.

\textbf{Behavioral surprise:} \textbf{FRO} is no longer unanimous: Strategy~B issues SELL on turbulence-style thresholds while A and C prefer HOLD---exactly the stress case this basket was meant to surface.

\section{Reflection}

For a one-month, \$10{,}000 experiment \textbf{after this run}, I would treat \textbf{Volatility Averse} as the primary risk brake on PLTR/FRO-style names where drawdown and turbulence triggers fired, but I would \textbf{not} automate Strategy~A's output without a JSON schema guard when news is empty---the TTE BUY rationale shows why. A simple hybrid could be: if \texttt{articles\_count = 0}, clamp Strategy~A to HOLD unless price momentum rules are explicitly enumerated and satisfied in code, not prose.
