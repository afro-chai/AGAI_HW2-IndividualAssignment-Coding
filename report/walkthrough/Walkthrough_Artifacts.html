Persistence: main.py

After strategies and the evaluator finish for a ticker, the pipeline writes structured JSON under outputs/: one file per symbol, then a run summary, then an optional backtest scorecard. Nothing here is LLM-authored source code—only serialized run results you can diff, redact, and commit for grading.

Write sequence

asyncio.gather runs each ticker; inside each ticker, strategies + evaluator complete, then main.py writes <TICKER>.json. When all tickers finish, summary.json aggregates agreement counts. --backtest adds backtest.json using the same ticker list.

Solid arrows: always for a normal run. Dashed: optional bonus scorecard.

Committed files (open samples)

Links resolve from a repo clone; samples are representative of schema and keys, not a live trading feed.

  • outputs/<TICKER>.json Full record: market_data_summary, news_features, strategy_a|b|c, evaluator. Example uses PLTR.
  • outputs/summary.json Aggregate: total_agreements, total_disagreements, per-row flags after all tickers complete.
  • outputs/backtest.json Heuristic post-hoc scorecard; same --tickers list as the main run when flag is set.
  • outputs/README.md Notes on what is safe to commit and how samples were produced.

Hygiene

Vendor error strings may appear inside news_features before redaction; use scripts/ helpers or manual review before publishing. The AI use appendix documents disposition of source vs. runtime JSON text.
