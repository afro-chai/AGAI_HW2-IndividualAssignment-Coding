Four conceptual zones map to the running code path: CLI entry and ticker-level concurrency, a single shared market JSON per ticker, three isolated LLM strategy branches, then the evaluator and file outputs.

Jump to agent specs — each canvas lists IN/OUT, prompt path, and where results land in outputs/.

Strategy canvases

Zone 1 — Entry & concurrency

  • src/main.py: parses CLI, loads .env, optional --backtest.
  • Per ticker: semaphore-limited asyncio.gather across tickers; inside each ticker, another gather for three strategies.
asyncio.gather(run_parallel_analysis(t) for t in tickers)

Zone 2 — Data plane (no LLM)

  • build_market_payload(ticker) in market_data.py: yfinance summary + optional Alpha Vantage NEWS_SENTIMENT.
  • Output is one JSON-serializable dict passed as the user message body to every strategy.

Zone 3 — Three parallel LLM branches

  • orchestration._run_one_strategy × 3 with distinct system_message from strategy_news_sentiment.txt, strategy_volatility_averse.txt, strategy_moral_trader.txt.
  • Each returns StrategyStructured: decision, confidence, justification.
  • No agent reads another agent’s output during this phase.

Zone 4 — Evaluator & artifacts

  • run_evaluator([a,b,c]): task text branches on unanimity / graded pair agreement.
  • pattern_note is set deterministically from the three decisions (not LLM-invented).
  • Write outputs/<TICKER>.json; then summary.json; optional backtest.json via backtest.py.
